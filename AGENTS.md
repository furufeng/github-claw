## 记忆管理

```
memory/
  LONG_TERM.md   ← 长期事实：用户偏好、项目背景、重要决策
  LOG.md         ← 时序日志：每次任务的简要记录（日期 + 做了什么 + 结论）
```

**规则：**
- 长期事实 → 写入 `memory/LONG_TERM.md`（不频繁变动的内容）
- 任务记录 → 追加到 `memory/LOG.md`（每次完成任务后必须追加）
- 临时草稿 → 只在对话中存在，不写入文件

---

## 工作方式

1. **开始对话时**：读 `AGENTS.md` → 读 `memory/LONG_TERM.md` → 了解最新状态
2. **执行任务时**：优先用仓库已有的工具和约定；小步提交，保持可追踪
3. **遇到模糊需求时**：先做出合理假设并说明，再询问确认
4. **遇到超出范围的请求时**：说明原因，提供替代方案

## 每次完成任务后的收尾动作

完成任何有意义的任务后，必须执行以下收尾步骤：

- [ ] 将任务摘要追加到 `memory/LOG.md`（格式见下方）
- [ ] 如果发现了新的长期事实或用户偏好，更新 `memory/LONG_TERM.md`
- [ ] 提交所有改动到仓库（包括记忆文件）
- [ ] 在对话中简短告知用户：做了什么、写了什么、下次可以从哪里继续

### LOG.md 条目格式

```
## YYYY-MM-DD — [任务简述]

**做了什么：** ...
**结论/产出：** ...
**待跟进：** ...（可选）
```

## 扩展约定

- 新增工具或脚本请放在 `tools/` 目录
- 新增项目或专题请放在 `projects/<名称>/` 目录
- 项目级技能统一保存在 `.agents/skills/`
- 每个技能必须使用独立目录：`.agents/skills/<skill-name>/`
- 所有文件使用 UTF-8，Markdown 格式优先
- 不在仓库中存放敏感信息（密钥、密码等）

## AI 技能机制（发现 / 安装 / 使用）

为保证主要 AI 代理在不同会话下都能稳定工作，统一采用以下机制。

### 1) 技能发现（Discovery）

1. 只扫描 `.agents/skills/` 下的一级子目录作为候选技能
2. 每个技能目录名使用小写字母、数字和中划线（如 `repo-audit`）
3. 每个技能目录必须包含 `SKILL.md`，用于描述技能名称、用途、输入输出和依赖

### 2) 技能安装（Install）

每个技能目录必须包含 `install.sh`，并满足：

- 可重复执行（幂等）
- 失败即退出（`set -euo pipefail`）
- 仅安装该技能所需的最小依赖

### 3) 技能使用（Use）

每个技能目录必须包含 `use.md`，明确：

- 何时调用该技能
- 调用前置条件
- 最小调用示例与预期输出
- 常见失败与排查方式

### 4) 稳定性保障

- 通过 GitHub Actions 工作流持续校验上述目录与文件约定
- 任何不满足约定的变更不得进入主分支
